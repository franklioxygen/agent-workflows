# 性能评审清单

## 常见风险区域

- 对无界集合进行嵌套循环。
- 在循环中发起数据库查询或逐项 API 调用。
- 缺少分页、限制、索引或过滤条件。
- 全表扫描、宽连接和无界排序。
- 将大 payload 一次性完整载入内存。
- 在请求路径上执行同步 I/O。
- 重复序列化、解析、正则处理或高成本计算。
- 缓存缺少失效策略或缺少用户/租户隔离。

## 验证选项

- 使用大输入的单元测试或集成测试。
- 查询 explain plan 或索引检查。
- 面向 CPU 密集逻辑的微基准测试。
- 面向请求路径的压测或冒烟基准测试。
- 面向内存、CPU 或 I/O 的性能剖析跟踪。
- 上线后的指标检查：延迟、错误率、队列积压、内存、CPU、数据库负载。

## 评审输出

- 说明预期规模和最坏情况。
- 解释为什么这个代码路径是热点，或者为什么它是有界的。
- 给出准确的验证命令，或说明跳过验证的原因。
- 把性能阻塞项与可选优化建议区分开。
