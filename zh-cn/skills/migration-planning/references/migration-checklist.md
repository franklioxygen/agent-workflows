# 迁移规划清单

## 兼容性

- 在发布重叠窗口内，新代码既能运行在旧数据上，也能运行在新数据上。
- 旧代码能够容忍新增的数据或字段。
- 公共 API 保持向后兼容，或提供版本化/废弃路径。
- 事件和队列可以容忍新旧生产者与消费者混跑。
- 特性开关以安全失败方式工作，并且有明确的移除负责人。

## 数据安全

- 回填操作具备幂等性、可恢复性、批处理能力和限速能力。
- 大表变更避免长时间锁和整表重写。
- 破坏性步骤具备备份、验证和明确审批。
- 验证检查覆盖行数、约束、可空性、重复值和引用完整性。

## 发布与回滚

- 每个阶段都有前置检查、执行步骤、验证、监控和回滚方案。
- 除非明确接受，否则回滚不应依赖从备份恢复。
- 可观测性覆盖错误、延迟、数据漂移、队列积压和用户影响。
- 最终清理只在兼容窗口关闭后进行。
