# 共享技能执行规则

## 安全

- 未经明确许可，不要提交或推送。
- 未经明确许可，不要部署、发布、创建标签、轮换密钥、吊销凭证、修改生产数据或执行破坏性命令。
- 对于评审、规划、审计和文档分析任务，除非用户明确要求编辑，否则保持只读。

## 验证

- 当仓库路径或文件路径可用时，运行该技能对应的确定性辅助脚本。
- 在修改工作流文档、技能元数据、共享参考资料或 README 清单后，运行 `skills/workflow-maintainer/scripts/audit_workflow_library.py`。
- 报告运行过的命令、结果、跳过的验证以及剩余风险。

## 范围

- 让修改严格限定在所选技能对应的任务范围内。
- 如果任务跨入实现、缺陷修复、功能设计、迁移执行、版本发布或事故响应，明确交接到更具体的工作流或技能。
- 对重复规则优先使用共享参考资料，对领域专属流程优先使用本地技能文档。
