# 安全评审清单

## 认证与会话处理

- 认证来源可信且无法被伪造。
- 会话、令牌和 Cookie 处理符合项目约定。
- 如果改动涉及登录、登出、刷新、模拟登录或服务账号路径，均已被纳入考虑。
- 错误信息不会在非预期情况下暴露凭证是否有效或账户是否存在。

## 授权与租户隔离

- 每个受影响入口点都有授权决策。
- 对象归属、租户边界和角色检查在服务端强制执行。
- 批量操作和列表接口不会绕过逐对象检查。
- 管理员、客服和后台任务路径不会继承到非预期权限。

## 输入、注入与执行

- 已检查 SQL、NoSQL、命令、路径、模板、HTML、重定向和 header 等危险汇点。
- 用户可控字符串在进入汇点前已被参数化、转义、规范化或拒绝。
- 文件上传路径校验了类型、大小、扩展名、内容、存储路径和访问策略。
- 反序列化和动态代码执行被避免，或被严格限制。

## 密钥与敏感数据

- 密钥不会被硬编码、记录日志、返回给客户端、提交进仓库或通过错误暴露。
- PII 和凭证会在日志、指标、trace、分析系统和第三方调用中被脱敏。
- 传输中和静态存储中的敏感数据遵循项目约定。
- 测试夹具不包含真实凭证或生产数据。

## 依赖与配置

- 新增或升级的依赖是必要的、被维护的，并且不属于已知高危漏洞依赖。
- 安全相关默认值显式设置：CORS、TLS 校验、debug 模式、cookie 标志、CSP、限流等。
- 特性开关和发布门在适用时以“默认关闭”方式失效。

## 评审输出

- 发现项包含可利用或失败场景，而不是泛泛的风险描述。
- 严重级别与真实可利用性和影响相匹配。
- 已报告验证命令和跳过的检查。
- 即使没有发现问题，也要说明剩余风险。
