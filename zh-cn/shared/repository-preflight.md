# 共享仓库预检提示词

说明：`agent-workflows` 中各个工作流文档共享的、可复用的仓库感知型预检提示词。

使用下面与场景匹配的预检块，而不是在每个工作流文件里重复复制同样的仓库检查内容。

<a id="standard-coding-preflight"></a>

## Standard Coding Preflight（标准编码预检）

用于功能开发、缺陷修复、重构和技术债清理。

```text
开始工作前先检查仓库。

你的目标：

1. 阅读适用于当前目录的仓库本地说明，例如 `AGENTS.md`、`CLAUDE.md`、`CONTRIBUTING.md`，或等效的工作流文档。
2. 检查当前分支和工作区状态。
3. 识别与当前任务无关的未提交变更。
4. 记录仓库中关于分支策略、测试、审批、代码风格或文件归属的规则。
5. 说明你将如何避免覆盖、回退或重新格式化与本任务无关的用户变更。
6. 确认在获得明确许可之前不会提交或推送。

请提供：

- 找到的适用仓库说明
- 当前分支 / 工作区状态
- 需要避免触碰的无关本地变更
- 会影响此任务的工作流约束

未经我明确许可，不要提交或推送。
除非我明确要求，否则不要回退或覆盖不是你创建的变更。
```

<a id="code-review-preflight"></a>

## Code Review Preflight（代码评审预检）

用于评审 Pull Request、分支或工作区 diff。

```text
开始评审前先检查仓库。

你的目标：

1. 阅读适用于当前目录的仓库本地说明，例如 `AGENTS.md`、`CLAUDE.md`、`CONTRIBUTING.md`，或等效的工作流文档。
2. 检查当前分支和工作区状态。
3. 明确评审目标和比较基线：
   - 如果提供的是 GitHub PR 链接，从 PR 元数据里识别其基线分支。
   - 如果提供的是分支名，结合仓库约定和 merge-base 信息判断最可能的基线分支。
   - 如果评审的是工作区变更，判断评审范围是整个工作区还是其中某个文件子集。
4. 识别与评审目标无关的未提交变更。
5. 记录仓库中关于分支、测试、审批、代码风格、所有权或评审预期的规则。
6. 如果评审目标或比较基线仍然有歧义，先停下来提问，再继续。
7. 确认在获得明确许可之前不会改代码、提交或推送。

请提供：

- 找到的适用仓库说明
- 当前分支 / 工作区状态
- 评审目标和比较基线
- 需要避免与评审目标混淆的无关本地变更
- 会影响此次评审的工作流约束

未经我明确许可，不要改代码、提交或推送。
除非我明确要求，否则不要回退或覆盖不是你创建的变更。
```

<a id="incident-response-preflight"></a>

## Incident Response Preflight（事故响应预检）

用于生产事故或事故后的排障工作流。

```text
开始工作前先检查仓库。

你的目标：

1. 阅读适用于当前目录的仓库本地说明，例如 `AGENTS.md`、`CLAUDE.md`、`CONTRIBUTING.md`、`RUNBOOK.md`，或等效的工作流与事故文档。
2. 检查当前分支和工作区状态。
3. 识别与此次事故无关的未提交变更。
4. 记录仓库中关于分支、测试、审批、热修流程或部署的规则。
5. 识别可用的可观测性工具：代码库或文档中提到的日志、监控看板、错误跟踪、APM 或告警系统。
6. 说明你将如何避免覆盖、回退或重新格式化与本次事故无关的用户变更。
7. 确认在获得明确许可之前不会提交、推送、部署或执行破坏性命令。

请提供：

- 找到的适用仓库说明
- 仓库中记录的热修或事故处理流程（如果有）
- 当前分支 / 工作区状态
- 需要避免触碰的无关本地变更
- 已识别的可观测性工具和看板

未经我明确许可，不要提交、推送或部署。
除非我明确要求，否则不要回退或覆盖不是你创建的变更。
```
